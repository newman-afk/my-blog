`createContext` lets you create a context that components can provide or read.

```jsx
const SomeContext = createContext(defaultValue)
```

[官方文档链接](https://react.dev/reference/react/createContext)

## SomeContext.Consumer

`createContext`与`useContext`配合使用，but Before useContext existed, there was an older way to read context:

```jsx
function Button() {
  // 🟡 Legacy way (not recommended)
  return <ThemeContext.Consumer>{(theme) => <button className={theme} />}</ThemeContext.Consumer>
}
```

Although this older way still works, but newly written code should read context with `useContext()` instead:

```jsx
function Button() {
  // ✅ Recommended way
  const theme = useContext(ThemeContext)
  return <button className={theme} />
}
```
